General

¿Qué es esta aplicación?

Es un sistema pensado para gestionar pacientes, aplicar encuestas clínicas y generar reportes, sin depender de papel ni de planillas sueltas.

Con la aplicación, el equipo puede:

  • Registrar y administrar pacientes.
  • Aplicar encuestas y tests clínicos (19 instrumentos distintos: depresión, ansiedad, calidad de vida, memoria, hábitos, entre otros).
  • Agrupar pacientes y encuestas dentro de investigaciones/estudios, con su consentimiento informado.
  • Generar reportes con gráficos, y descargarlos en PDF o Excel.
  • Crear encuestas propias y personalizadas, sin que nadie tenga que programar.
  • Seguir funcionando aunque no haya internet, sincronizando los datos automáticamente cuando la conexión vuelve.

Funciona tanto en tablets/celulares Android como en una computadora, desde el navegador web — con la misma información sincronizada entre ambos.

General

¿Con qué está construida?

Toda aplicación moderna se apoya en distintas piezas de tecnología, cada una con un trabajo específico. Acá va la lista, explicada con analogías simples.

Flutter Lenguaje / herramienta

Es la herramienta con la que se construyó la aplicación. Su ventaja principal es que se escribe una sola vez y funciona en varios lugares: celular (Android) y computadora (Web), sin duplicar el trabajo cada vez que se corrige algo. Es una tecnología creada por Google y usada por empresas grandes a nivel mundial, lo que da estabilidad a largo plazo.

Supabase Nube / datos

El servicio que guarda la información de forma segura en internet: pacientes, encuestas, respuestas y usuarios. Es como una caja fuerte digital compartida. También maneja el inicio de sesión, para que solo personal autorizado entre al sistema.

Hive Guardado local

Permite que la app siga funcionando sin conexión a internet. Los datos se guardan en el dispositivo y, apenas vuelve la conexión, se sincronizan automáticamente con la nube. Evita perder información por cortes de señal.

GitHub Control de versiones

El "historial de cambios" del proyecto. Guarda cada modificación hecha al sistema, quién la hizo y cuándo, y permite revisar los cambios antes de que lleguen a la versión en uso.

Vercel Publicación web

El servicio que publica automáticamente la versión web del sistema en internet, cada vez que el equipo aprueba una mejora. Nadie sube archivos a mano: el proceso es automático.

Reportes PDF / Excel Exportación

La app transforma resultados de encuestas en gráficos, PDF listos para imprimir o adjuntar a una historia clínica, y Excel para quienes prefieran analizar los datos en planilla.

Lectura en voz alta Accesibilidad

Algunas encuestas pueden leerse en voz alta desde la propia app, pensado para pacientes con dificultades de lectura o visión. Disponible solo en la versión para celular/tablet.

Detección de conexión Conectividad

La app sabe en todo momento si hay o no internet, y muestra indicadores visuales de qué datos están sincronizados y cuáles quedan pendientes de subir a la nube.

Uso diario

Flujo de sincronización

Así es, paso a paso, cómo la app protege el trabajo diario incluso sin internet.

1. Se completa un paciente o una encuesta

Con o sin conexión a internet, el trabajo se puede realizar con normalidad.

2. Guardado local automático

Los datos quedan guardados en el propio dispositivo (Hive), sin depender de la red.

3. Se recupera la conexión

Cuando el dispositivo vuelve a tener internet, la app lo detecta automáticamente.

4. Sincronización con la nube

Los datos pendientes se suben a Supabase. También existe un botón para sincronizar manualmente cuando se desee.

5. Datos confirmados

La app muestra un indicador visual de que ya no quedan datos pendientes por sincronizar.

Recomendación

Antes de cerrar la app al final del día, conviene revisar que no queden datos marcados como "pendientes de sincronizar", sobre todo si se va a usar otro dispositivo a continuación.

Actualizaciones

¿Dónde se actualiza y publica el sistema?

Acá entran GitHub y Vercel, dos herramientas invisibles para el usuario final, pero que hacen posible que el sistema mejore con el tiempo de forma ordenada y segura.

GitHub — el "historial de cambios" del proyecto

Es como un Google Drive especializado para código, pero con superpoderes:

  • Guarda todo el historial de cada cambio: qué se modificó, quién lo hizo y cuándo.
  • Permite que el equipo trabaje ordenado y sin pisarse el trabajo: cada mejora se prueba por separado antes de sumarse a la versión oficial.
  • Sirve como respaldo permanente: si algo se rompe, siempre se puede volver a una versión anterior que funcionaba bien.

El equipo trabaja con una "versión oficial en uso" y "versiones de prueba". Los cambios nuevos solo pasan a producción después de ser revisados por al menos otra persona del equipo, lo que reduce mucho el riesgo de que un error llegue a la versión en uso.

Vercel — la publicación automática de la versión web

Vercel es el servicio que hospeda y publica automáticamente la versión web de la aplicación (la que se usa desde el navegador, sin instalar nada).

1. Equipo dev

Desarrolla y prueba una mejora

2. GitHub

Revisión y aprobación del cambio

3. Vercel

Publica la web automáticamente

4. Usuarios

Ven la versión nueva al entrar

En resumen

GitHub es el lugar donde se controla y revisa el código, y Vercel es el servicio que toma esa versión aprobada y la publica automáticamente en internet. Es un proceso completamente automatizado: nadie "sube archivos a mano".

¿Y la versión para Android?

La versión para celular/tablet se genera como un archivo instalable (APK) a partir de esa misma versión oficial del código, y se comparte para su instalación cuando corresponde a una nueva entrega.

Seguridad

Seguridad y privacidad de los datos

El sistema es de acceso restringido

No es de acceso público: se necesita usuario y contraseña asignados por el equipo administrador.

  • La información de pacientes se guarda en un servicio en la nube (Supabase) con reglas de seguridad que restringen quién puede ver o modificar cada dato.
  • Las claves técnicas sensibles (las que permitirían administrar la base de datos completa) nunca se incluyen en la aplicación ni en el navegador; solo se usan del lado seguro del servidor.
  • Todo cambio de código pasa por una revisión de al menos otra persona del equipo antes de llegar a la versión en uso.
  • El acceso queda registrado por usuario, lo que permite trazabilidad de quién hizo qué dentro del sistema.
Seguridad

Buenas prácticas de uso

Recomendaciones simples para el equipo del consultorio, que ayudan a mantener los datos seguros y ordenados.

  • Cerrar sesión al terminar de usar la app en dispositivos compartidos o públicos.
  • No compartir usuario ni contraseña entre distintas personas del equipo.
  • Antes de terminar la jornada, revisar que no queden datos marcados como "pendientes de sincronizar".
  • Si se usa la app en varios dispositivos, sincronizar seguido para evitar duplicados o información desactualizada.
  • Mantener la aplicación actualizada a la última versión entregada por el equipo de desarrollo.
  • Exportar reportes (PDF/Excel) periódicamente si se necesitan respaldos locales adicionales.
  • Ante cualquier comportamiento extraño del sistema, avisar al equipo técnico en vez de intentar "arreglarlo" manualmente.
Ayuda

Resolución de problemas

Pasos simples para los inconvenientes más comunes, antes de contactar al equipo técnico.

No puedo iniciar sesión
  1. Verificar que el usuario y la contraseña estén bien escritos.
  2. Verificar que haya conexión a internet (necesaria para iniciar sesión).
  3. Si el problema continúa, puede que el usuario no exista o haya sido dado de baja: contactar al equipo técnico.
La app no sincroniza
  1. Revisar que el dispositivo tenga conexión a internet.
  2. Revisar el indicador de sincronización pendiente dentro de la app.
  3. Esperar unos segundos y usar el botón manual de sincronización.
  4. Si el problema persiste, contactar al equipo técnico sin borrar datos.
La pantalla queda en blanco o no carga (versión web)
  1. Recargar la página del navegador.
  2. Probar con Chrome o Edge actualizados.
  3. Borrar la caché del navegador y volver a intentar.
  4. Verificar la conexión a internet.
No encuentro un paciente o una encuesta
  1. Revisar si hay algún filtro o texto de búsqueda activo.
  2. Confirmar que el registro se guardó correctamente en su momento.
  3. Revisar el estado de sincronización del dispositivo donde se creó.
  4. Verificar que se inició sesión con el usuario correcto.
El PDF o Excel no se descarga
  1. Revisar si el navegador está bloqueando ventanas emergentes o descargas.
  2. Verificar que haya espacio de almacenamiento disponible.
  3. Intentar nuevamente o probar con otro navegador.
Olvidé mi contraseña

No existe un registro abierto al público, por lo que hay que contactar al equipo técnico o administrador del sistema para restablecerla de forma segura.

¿Sigue sin resolverse?

Si el problema persiste después de estos pasos, contactar al equipo de desarrollo/soporte técnico del sistema, describiendo qué se intentó y en qué dispositivo ocurre.

Ayuda

Preguntas frecuentes

¿Si se corta internet en el consultorio, se pierde el trabajo?

No. La app sigue funcionando y guardando todo localmente. Cuando vuelve la conexión, sincroniza sola.

¿Quién puede ver los datos de los pacientes?

Solo las personas que tengan un usuario y contraseña creados específicamente para el equipo autorizado.

¿Cómo se actualiza la aplicación?

La versión web se actualiza sola, automáticamente, cada vez que el equipo publica una mejora aprobada. La versión Android se actualiza instalando un nuevo archivo cuando el equipo entrega una nueva versión.

¿Se puede perder información al actualizar?

No debería: los cambios se prueban en un ambiente separado antes de aplicarse a la versión oficial, y todo el historial de versiones queda guardado por si hiciera falta volver atrás.

¿Necesito instalar algo para usar la versión web?

No. Solo se necesita un navegador (se recomienda Chrome o Edge) y conexión a internet la primera vez.

¿Puedo usar la app en varios dispositivos a la vez?

Sí. Mientras cada dispositivo tenga conexión para sincronizar, los datos se mantienen consistentes en la nube.

¿Los pacientes pueden acceder al sistema?

No. El sistema es de uso exclusivo del personal autorizado.
